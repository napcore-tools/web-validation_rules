# Notes

Certain aggregation of child elements include an order attribute (which can be used to qualify the id) . - Normally child elements should be serialised in the same sequence as the order attribute. Specific examples are POINTS IN PATTERN. CALLs. - Values for the order attribute should be unique within parent. Upstream numbering clash: C.02.03 is assigned to 2 distinct rules in the source sheet; local -b/-c suffixes keep records apart.
