# Notes

Certain aggregation of child elements include an order attribute (which can be used to qualify the id) . - Normally child elements should be serialized in the same sequence as the order attribute. Specific examples are POINTS IN PATTERN, CALLs, TRAIN COMPONENTs, etc.. Code normalized from 'c.04.02' in the source sheet.
